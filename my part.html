Sound & Pressure Safety Module

This website presents my personal contribution to the Smart Blind Shoes project: a pressure-based slip detection system with real-time sound feedback.

Project Overview

The Smart Blind Shoes project aims to improve user safety through assistive feedback. While the full system includes navigation and emergency features, this page focuses exclusively on my safety-critical module.

Pressure Sensor Integration (HX711)

  • Integrated HX711 load cell amplifier with foot pressure sensor
  • Real-time pressure sampling and processing
  • Prepared for calibration and scaling in real hardware
  • Pressure used as an indicator of ground contact stability

Slip Detection Logic

  • Detection of sudden pressure drops
  • Minimum pressure threshold monitoring
  • Timeout-based loss of stable contact detection
  • Designed as a safety-triggering algorithm

Sound Feedback System

  • Distinct sound patterns for different events
  • Slip alert: clear and recognizable audio signal
  • Emergency alarm with low-frequency rhythm
  • Audio feedback designed for instant user awareness

Non-Blocking Audio Logic

  • No delay() usage in sound generation
  • millis()-based scheduling
  • Ensures continuous sensor reading and system responsiveness
  • Supports simultaneous sensing and feedback

Pressure Data Visualization

  • Real-time pressure graphs using Serial Plotter
  • Threshold line visualization
  • Slip events shown as clear spikes
  • Used for validation and debugging

Why a Separate Demo Project?

The full system uses many peripherals (ultrasonic sensors, LCD, RGB, GPS, vibration motors). In Wokwi and on Arduino Uno, this leads to pin and timing limitations.

To ensure accurate HX711 sampling, clean graphs, and stable non-blocking audio behavior, I demonstrate my module as a standalone project.

Live Demonstration

  • Open Serial Plotter
  • Show pressure signal, threshold, and slip spike
  • Trigger slip → observe audio alert
